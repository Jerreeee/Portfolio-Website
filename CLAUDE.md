This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Personal portfolio website built with Next.js 16 (App Router) using static site export. Showcases projects with rich media (images, videos, code samples), an interactive resume viewer, and an about page.

## Commands

- `npm run dev` — Start dev server (Turbopack)
- `npm run build` — Production build (static export to `out/`)
- `npm run lint` — ESLint
- `npm run generate:all` — Regenerate all auto-generated files (project manifests, component classes/indexes, theme augmentation, icons)
- `npm run generate:projects` — Regenerate project manifests and index
- `npm run generate:components` — Regenerate component classes, index, animation types, and settings types
- `npm run generate:resume` — Generate resume PDF via Puppeteer
- `npx serve out` — Serve the static export locally

Scripts in `scripts/` run via `ts-node` with their own tsconfig at `scripts/tsconfig.scripts.json`.

## Architecture

### Routing (App Router)

Two route groups under `src/app/`:
- `(site)/` — Main pages: projects list, project detail (`[slug]`), about, resume
- `(raw)/` — Raw layouts for tooling (e.g., `/resume/raw` used by Puppeteer for PDF generation)

The home page (`/`) redirects to `/projects`. Output mode is `export` (fully static), with `trailingSlash: true` and `images.unoptimized: true`.

### Theme & Component System

Components live under `src/Themes/Default/Components/`. Each component follows a slot-based styling pattern:

1. `createUtilityClasses()` generates CSS class names for slots
2. `makeSlot()` (in `src/Utils/makeSlot.ts`) creates styled slot components using MUI's `styled()` with Emotion, automatically wiring up theme-defined slot animations and class names
3. Components register into the MUI theme via `theme.components` for slot animations and settings

The theme registry is in `src/Themes/index.ts`. Currently only `Default/Dark` is implemented. Theme augmentation types are auto-generated at `src/Types/Overrides/mui-theme-augmentation.d.ts`.

### Data Layer

**Projects:** Each project is a directory under `src/Data/Projects/[ProjectName]/` containing:
- `data.ts` — Project metadata (slug, title, descriptions, technologies, links)
- `ProjectCmp.tsx` — Custom React component for project detail page
- `manifest.ts` — **Auto-generated** media metadata (image dimensions, video info)

Project display order is controlled by `src/Data/Projects/projectOrdering.ts`. The barrel file `src/Data/Projects/index.ts` is auto-generated by `generate:projects-index`.

**Static assets** for projects live at `public/projects/[ProjectName]/{Images,Videos,Code,Downloads}/`.

### Path System

`src/Config/paths.ts` exports `PATHS` — a set of path builders that return `PathObject` instances. These convert between filesystem (`.fs()`), URL (`.url()`), and import (`.import()`) path formats, keeping path references consistent across build scripts and runtime code.

### Code Generation

Multiple scripts in `scripts/` auto-generate TypeScript files. Files marked as auto-generated should not be manually edited:
- Project manifests and index (`src/Data/Projects/*/manifest.ts`, `src/Data/Projects/index.ts`)
- Component class definitions, barrel exports, animation types, settings types
- MUI theme augmentation types
- Icon manifests and transformed SVGs

Run `npm run generate:all` after adding new projects, components, or icons.

### Key Patterns

- **Imports:** Use `@/*` path alias (maps to `src/`). Case-sensitive imports enforced by ESLint.
- **Styling:** Combination of MUI `styled()` via `makeSlot`, Emotion CSS-in-JS, and Tailwind CSS 4.
- **Animations:** Motion (Framer Motion) library. Shared animation definitions in `src/Themes/animations.ts`.
- **Formatting:** Prettier with single quotes, semicolons, trailing commas, 2-space indent, 80 char width.
