<h1 class="has-luminous-vivid-orange-background-color has-background has-medium-font-size wp-block-heading" id="introduction">Introduction</h1>



<h4 class="has-background wp-block-heading" id="about-me" style="background-color:#333333">About me</h4>



<p id="viewer-ef5k0">Hello, my name is Jeroen Denayer and I&#8217;m from Belgium. I am a Technical Artist currently studying a <a href="https://www.digitalartsandentertainment.be">Digital Arts &amp; Entertainment program</a> at the Howest university of applied sciences for my Bachelor&#8217;s Degree. </p>



<p>Why did I choose Python in Houdini? </p>



<p>The game industry is all about speeding up game development. Games are becoming bigger, more qualitative. Companies are met with ever increasing pressure from tight deadlines to get new updates out for the player. And of course getting as many sales as possible. This is made possible thanks to amazing tool developers who create easy to use, highly interactive tools that update real-time in a game engine when the user changes something. This keeps the user from going back to their modelling software &gt; changing an asset &gt; exporting to disk and reimporting into the game engine. My personal interests are completely in line with this mindset of automating highly repetitive work while still focusing on art <span class="has-inline-color has-black-color">directability</span>. Everything related to scripting, proceduralism and getting your assets inside of a game engine as fast as possible is a great interest of mine. That&#8217;s why I wanted to research all the steps from creating a tool to in the end having an artist use the tool inside of a game engine.</p>



<h4 class="has-background wp-block-heading" id="what-to-expect" style="background-color:#333333">What to expect?</h4>



<p id="viewer-1juh9">As a practical proof point I have created a Houdini Digital Asset (HDA) that helps in the creation of a procedural racetrack that can be integrated into Unreal Engine. Instead of updating parameters of the Houdini Asset in Unreal, I will make use of Houdini session sync and python viewer states to change parameters of the HDA in Houdini. This will allow us to make changes to the racetrack inside of Houdini and see it automatically update in Unreal as well.</p>



<p>Here is what I will cover:</p>



<ul class="wp-block-list"><li>How to interact with objects in your viewport using your mouse and keyboard.</li><li>How to make a super interactive UI that can be navigated using only your mouse and keyboard.</li><li>How to store all the data of the entire racetrack in one json.</li><li>How we can get this json into our geometry network so we can construct the racetrack mesh from it using regular Houdini nodes..</li><li>Why I made the decision to use certain features such as Houdini session sync and python viewer states. </li><li>How to integrate this tool into Unreal Engine</li></ul>



<p>I will not cover:</p>



<ul class="wp-block-list"><li>How to use this json data to create the racetrack mesh itself</li></ul>



<div style="height:100px" aria-hidden="true" class="wp-block-spacer"></div>



<h1 class="has-black-color has-luminous-vivid-orange-background-color has-text-color has-background has-medium-font-size wp-block-heading" id="literature-prior-art-study">Literature/prior-art study </h1>



<h4 class="has-background wp-block-heading" id="creating-racetracks" style="background-color:#333333">Creating racetracks</h4>



<p>Starting off with looking for what&#8217;s already been done in this field I found some interesting sources which helped me with my research. Let&#8217;s start off with the creation of the procedural racetrack itself. There are a couple of techniques that are commonly used to create racetracks for games.</p>



<div class="wp-block-media-text alignwide has-media-on-the-right is-stacked-on-mobile is-vertically-aligned-bottom"><figure class="wp-block-media-text__media"><img decoding="async" width="1024" height="470" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/TrackmaniaTrackEditor-1024x470.jpg" alt="" class="wp-image-35 size-full" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/TrackmaniaTrackEditor-1024x470.jpg 1024w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/TrackmaniaTrackEditor-300x138.jpg 300w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/TrackmaniaTrackEditor-768x352.jpg 768w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/TrackmaniaTrackEditor-1536x705.jpg 1536w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/TrackmaniaTrackEditor.jpg 1920w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure><div class="wp-block-media-text__content">
<h5 class="has-background wp-block-heading" id="pre-made-segments" style="background-color:#2b2b2b">Pre-made segments</h5>



<p>In this approach you create pre-made segments of the track (corner segment, straight segment, etc&#8230;) and then place one segment after the other to create the track. The Trackmania games make use of this approach. The advantage is that different tracks are very consistent in their style as they consist of the same pre-made segments. The disadvantage is that you loose the ability to create very specific turns like 17.6 degrees to the left as all the segments have pre-defined standard angles like 45 or 90 degrees.</p>
</div></div>



<div class="wp-block-media-text alignwide has-media-on-the-right is-stacked-on-mobile is-vertically-aligned-bottom"><figure class="wp-block-media-text__media"><img loading="lazy" decoding="async" width="600" height="338" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/Splinemesh.gif" alt="" class="wp-image-34 size-full"/></figure><div class="wp-block-media-text__content">
<h5 class="has-background wp-block-heading" id="bending-pre-made-segments" style="background-color:#2b2b2b">Bending pre-made segments</h5>



<p>A second option is to create pre-made segments that have multiple edge loops along their length and then bend them along a curve inside of a game engine. This approach is often used for cables inside of games and gives you the freedom to turn in any direction. But it comes with some negatives. It is a quick and dirty way. These extra edge loops are wasted geometry on straight parts of the curve and this bending method can also cause pinching and stretching of UV&#8217;s at parts with a high curvature. </p>
</div></div>



<h5 class="has-background wp-block-heading" id="fully-procedural-racetrack" style="background-color:#2b2b2b">Fully procedural racetrack</h5>



<p id="viewer-enqvq">Using Houdini, the company Space Ape created a fully procedural racetrack based on an input curve that defines the main shape. In <a href="http://The following GDC talk (link) shows how the company Space Ape tackled this problem and how they ultimately started using the fully procedural method to create race tracks.">this</a> GDC talk, they explain all the benefits of using this method. It gives you full flexibility over the direction of your track, optimization of polycount on straight parts and automatically creating correct UV&#8217;s for your track. Their tool was directly implemented inside of a game engine and updates in real-time when a change is made. This is thanks to the Houdini Engine for Unreal plugin which connects both software.  It allows you to change parameters inside of Unreal Engine and in the background Houdini will re-calculate the output. Space Ape created their game in Unity but everything is applicable to Unreal Engine as well.</p>



<figure class="wp-block-gallery alignwide has-nested-images columns-default is-cropped wp-block-gallery-2 is-layout-flex wp-block-gallery-is-layout-flex">
<figure class="wp-block-image size-full"><img loading="lazy" decoding="async" width="1920" height="1080" data-id="63" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image_2022-01-22_164602.png" alt="" class="wp-image-63" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-22_164602.png 1920w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-22_164602-300x169.png 300w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-22_164602-1024x576.png 1024w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-22_164602-768x432.png 768w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-22_164602-1536x864.png 1536w" sizes="auto, (max-width: 1920px) 100vw, 1920px" /><figcaption>Space Ape tool end result</figcaption></figure>



<figure class="wp-block-image size-full"><img loading="lazy" decoding="async" width="1836" height="915" data-id="48" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/UnityHoudiniEngine3.gif" alt="" class="wp-image-48"/><figcaption>Space Ape tool in action</figcaption></figure>
</figure>



<h4 class="has-background wp-block-heading" id="viewer-2dm7f" style="background-color:#333333">Limitations of Unreal and Houdini Engine</h4>



<p>The tool that Space Ape created was amazing! They could fully customize tons of parameters to change the visual look of the racetrack. However, to do this they still needed to drag around sliders in the UI. <mark style="background-color:rgba(0, 0, 0, 0)" class="has-inline-color has-black-color">This is the normal way of working when working with Unreal and Houdini. But this doesn&#8217;t fit in with the idea of making everything</mark> as interactive as possible for the artist that will use the tool in the end. What I want is something more like how you would model in Blender. Every action is done inside of the viewport using simple mouse clicks and keyboard shortcuts. And this is something that I wanted for my own racetrack tool as well. This is when I found out about Python viewer states.</p>



<h4 class="has-background wp-block-heading" id="viewer-e289m" style="background-color:#333333">Python viewer states</h4>



<p>Python viewer states are a way to interactively update parameter values on your tool or HDA using mouse clicks and keyboard shortcuts. </p>



<p>I found that some students from the Breda University of Applied Sciences <a href="https://youtu.be/mBdKK_VKDHQ?t=893">(link)</a> had already built something that was similar to what I wanted to do. I used their <a href="https://youtu.be/2XIIEhgcWMQ">tutorial</a> as a starting point for my own tool. They implemented some interactivity but I wanted to take this even a step further. I wanted to navigate the entire UI of my tool purely using python viewer states.</p>



<figure class="wp-block-image size-large"><img loading="lazy" decoding="async" width="1024" height="476" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/HoudiniTalkResult-2-1024x476.png" alt="" class="wp-image-55" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/HoudiniTalkResult-2-1024x476.png 1024w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/HoudiniTalkResult-2-300x140.png 300w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/HoudiniTalkResult-2-768x357.png 768w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/HoudiniTalkResult-2-1536x714.png 1536w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/HoudiniTalkResult-2.png 1879w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /><figcaption>Racetrack tool made by students from the Breda University of Applied Sciences </figcaption></figure>



<p>So using python viewer states we can now interactively work with our tool. But what is the benefit of using this approach if we can&#8217;t get our racetrack back into Unreal Engine? We can achieve this using <a href="https://www.youtube.com/watch?v=Yn9rnQRkCUQ">Houdini session sync</a>. This allows us to open a live connection between Unreal and Houdini and we can now jump back and forth between both softwares, change something in one and it will immediately update in the other and vice versa.</p>



<h4 class="has-black-color has-text-color has-background wp-block-heading" id="viewer-a5up1" style="background-color:#333333">Conclusion</h4>



<p>What was very clear to me, is the workflow/pipeline that I would use during my research. First create a tool inside of Houdini. Afterwards convert it into a digital asset which can be imported into Unreal engine. Then using Houdini engine and Houdini session sync we can open a live connection between the softwares and use python viewer states in Houdini to change parameters on our racetrack tool.</p>



<p id="viewer-8rrvr">Some questions that occurred to me after doing my literature study and before starting on my own tool were:</p>



<ol class="wp-block-list"><li>How does the tool compare when built with and without python viewer states? What are the pros and cons?</li><li>Is using Houdini engine in combination with Houdini session sync a benefit to the game asset creation pipeline?</li></ol>

<h1 class="alignwide has-text-align-left has-luminous-vivid-orange-background-color has-background has-medium-font-size wp-block-heading" id="workflow">Workflow</h1>



<h4 class="has-background wp-block-heading" id="python-module" style="background-color:#333333">Overview</h4>



<figure class="wp-block-image size-large"><img loading="lazy" decoding="async" width="1024" height="903" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/WorkflowDiagram-3-1024x903.png" alt="" class="wp-image-464" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/WorkflowDiagram-3-1024x903.png 1024w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/WorkflowDiagram-3-300x264.png 300w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/WorkflowDiagram-3-768x677.png 768w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/WorkflowDiagram-3-1536x1354.png 1536w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/WorkflowDiagram-3-2048x1805.png 2048w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/WorkflowDiagram-3-2000x1763.png 2000w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></figure>



<p>Here you can see an overview of how I connected all the different parts.</p>



<ul class="wp-block-list"><li>Step 1: create a standard procedural Houdini network using nodes.</li><li>Step 2: convert this network into a tool/HDA and import it into Unreal with the Houdini Engine plugin.</li><li>Step 3: change the position of curve points in Unreal Engine to shape the racetrack.</li><li>Step 4: use Houdini session sync to open a live connection between Unreal and Houdini.</li><li>Step 5: interactively work with the tool in the Houdini viewport using python viewer states.</li><li>Step 6: the viewer state will automatically update the UI when the artist changes something in the viewport.</li><li>Step 7: the python module will run when the artist changes a parameter value in the UI directly</li><li>Step 8: the only reason that this UI exists is to construct a json file that will hold all the data of the entire racetrack.</li><li>Step 9: json parameter and other parameters that will help us to construct the json.</li><li>Step 1: the procedural network will read in this json and construct a complete racetrack from it.</li></ul>



<h1 class="alignwide has-text-align-left has-luminous-vivid-orange-background-color has-background has-medium-font-size wp-block-heading" id="prototyping">Prototyping</h1>



<p id="viewer-7d0e8">This will be a more technical part as it is all about using Python. I will always mention all the functions that I created and explain their functionality. I will not always show the code as some of it is just Python basics. Making use of an external code editor is highly advisable but not necessary. I personally used Visual Studio Code. To interact with Houdini using python we will make use of the Houdini object model or <a href="https://www.sidefx.com/docs/houdini/hom/intro.html">HOM </a>for short. This is a python library that allows you to interact with Houdini. If you don&#8217;t now how something works, or how you can get access to a certain part of Houdini. Then HOM is your best friend. Thousands of functions, classes, objects and way more are all super well documented. Use this documentation!!</p>



<p>You will probably not understand everything I do the first time you read it or try it out yourself in Houdini. Be patient and dissect every step into smaller chunks. Begin by creating something small and make sure that it works perfect. Then you can start implementing complexer python systems to improve your tool.</p>



<p>Before creating a viewer state let&#8217;s first create our UI and the rest of our tool which we will link the viewer state to later on. Don&#8217;t worry, this also involves a lot of Python. I will skip the step on how to create the geometry network as you should already be able to do this if you read this. Let&#8217;s start with the basics of Python for Houdini digital assets and work our way up from there.</p>



<h4 class="has-background wp-block-heading" id="python-module" style="background-color:#333333">Python Module</h4>



<figure class="wp-block-image size-large is-resized m0"><img decoding="async" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/UIPythonModule-1-1024x629.png" alt="" class="wp-image-492" width="500" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/UIPythonModule-1-1024x629.png 1024w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/UIPythonModule-1-300x184.png 300w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/UIPythonModule-1-768x472.png 768w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/UIPythonModule-1-1536x943.png 1536w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/UIPythonModule-1.png 1632w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>



<p>The first thing that you should do when considering to enhance your tool using Python is learn about the Python Module. The Python Module is a type of event handler. It can run code whenever the user presses a button, manually enters a value or slides the slider around in the UI.  Doing this will trigger and event to which code can be linked in the Python Module. There are other event handlers like OnUpdated which we will be using as well. You can find these event handlers on your HDA under Type Properties &gt; Scripts.</p>



<p>How to link a parameter to a function in the python module? Start off by creating your function. Afterwards go to the parameter that you want to link this function to. In the parameter description under Callback Script we can call our function. </p>



<div class="wp-block-columns m0 is-layout-flex wp-container-core-columns-is-layout-9d6595d7 wp-block-columns-is-layout-flex">
<div class="wp-block-column is-layout-flow wp-block-column-is-layout-flow">
<p>Use <a href="https://www.sidefx.com/docs/houdini/hom/hou/phm.html">hou.phm()</a> to get a reference to the <a href="https://www.sidefx.com/docs/houdini/hom/hou/HDAModule.html">Python Module</a> and after that add the name of the function you created in the Python Module. Add kwargs as an argument. This is a dictionary from which you can get information about the current node, parameter the user changed, etc&#8230; . Make sure to set the script language to python.</p>
</div>

<h4 class="has-background wp-block-heading" id="json-vs-multiparameter-block" style="background-color:#333333">Json vs Multiparameter block</h4>



<figure class="wp-block-image size-large is-resized m0"><img decoding="async" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/UiGeometryNetwork-1024x714.png" alt="" class="wp-image-510" width="497" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/UiGeometryNetwork-1024x714.png 1024w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/UiGeometryNetwork-300x209.png 300w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/UiGeometryNetwork-768x535.png 768w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/UiGeometryNetwork.png 1531w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>



<p>Another super important step that we have to figure out is how we can transfer the data that defines our racetrack from our user interface into our geometry network so we can use regular Houdini nodes to construct the racetrack. But to figure that out we need to first figure out what this data looks like so we might have an idea on how to store it.</p>

<p>I will only define certain parameters at a couple of points on the racetrack. I will be calling these control handles or control points. The idea is that every point can contain multiple parameters that define how the racetrack looks and plays like at that point. Different points can have different parameters with different values. Looking in Houdini, I found two ways that are mostly used to store this data and then transfer it on to the geometry network.</p>



<div class="wp-block-columns m0 is-layout-flex wp-container-core-columns-is-layout-9d6595d7 wp-block-columns-is-layout-flex">

<div class="wp-block-column m0 is-layout-flow wp-block-column-is-layout-flow"></div>

<p>This is a special parameter type that allows you to create multiple instances of the same set of parameters. Every instance in this multiparameter would represent a control handle/point on our racetrack. This would allow us to define per point data (<a href="https://youtu.be/nS4FSMEas-I">example</a>).</p>

<h5 class="has-background wp-block-heading" id="pros" style="background-color:#2b2b2b">Pros</h5>



<ul class="wp-block-list"><li>Very visual and easy to understand</li><li>All parameters of all the handles are visible at the same time</li></ul>

<div class="wp-block-column is-vertically-aligned-top m0 is-layout-flow wp-block-column-is-layout-flow">
<h5 class="has-background wp-block-heading" id="cons" style="background-color:#2b2b2b">Cons</h5>



<ul class="wp-block-list"><li>Hard to manage using Python when handles are deleted or added</li><li>Every control handle will contain the exact same set of parameters. For&#8217; some of the control handles we probably only want to change specific parameters like the road width. This however is not possible when using a multiparameter. Every point contains all the parameters. This clutters the UI and is harder to manage.</li></ul>

<h5 class="has-background wp-block-heading" id="json" style="background-color:#2b2b2b">Json</h5>

<div class="wp-block-column is-layout-flow wp-block-column-is-layout-flow"></div>

<p>Using jsons is the other way to transfer data from the user interface to your geometry network.  A json allows you to put many different parameters in a single text field. Each element in the json is a dictionary and represents a control handle/point. I will be calling this the handle dictionary.  In this dictionary we can define all the per point data using key-value pairs. </p>

<h5 class="has-background wp-block-heading" id="pros" style="background-color:#2b2b2b">Pros</h5>



<ul class="wp-block-list"><li>Easy to manage through python</li><li>You can add handles that only change certain parameters but not others. So the json would only be as big as it needs to be.</li><li>Can save to disk. This would be a super easy way to transfer a racetrack between multiple artist who can work on it.</li></ul>
</div>



<div class="wp-block-column is-vertically-aligned-top is-layout-flow wp-block-column-is-layout-flow">
<h5 class="has-background wp-block-heading" id="cons" style="background-color:#2b2b2b">Cons</h5>



<ul class="wp-block-list"><li>Not as visual and easy to understand as a multiparameter </li><li>Editing a json structure manually is possible but error prone. Therefore, you need another way for the user to change parameter values in the json without having the user edit the json directly.</li></ul>
</div>

<h5 class="has-background wp-block-heading" id="conclusion" style="background-color:#2b2b2b">Conclusion</h5>
</div>



<div class="wp-block-column is-layout-flow wp-block-column-is-layout-flow"></div>
</div>



<p>The multiparameter block is a super nice Houdini feature but it has it&#8217;s drawbacks. When we use json on the other hand the possibilities to store our racetrack data are just endless as we are only limited by what Python can do. And python can a lot. So for my tool I decided to work with json to transfer data from the user interface to the geometry network. As I mentioned in the cons about using json, we don&#8217;t want an artist to manually change the json. We will need to create a user interface for this so the artist who will use the tool does not have to touch the json to change parameter values.</p>

<p>So what is this handle type? The handle type defines what parameters can be changed on that handle. I have a handle type &#8220;Normal&#8221; where you can only change the road width and rotation. Another handle type &#8220;Delete&#8221; which allows you to cut away parts of the racetrack. And the last one &#8220;Railings&#8221; which allows you to add some sort of barrier at the two sides of the track. Because I also added this handle type parameter, I need a way to only show the parameters of the chosen handle type. I set the &#8220;Hide When&#8221; value of the parent folders (Basic, Delete, Rail) to hide the folder when the handle type is 0, 1 or 2.</p>



<p class="m0">Below you can see that only the parameters from one handle type are shown and that all the other parameters are hidden. Note how the json also reflects these changes. I do this with some extra case handling in my code. I delete all the parameters that do not have the correct handle type. I can easily check if the parameter is from a certain handle type because I added Basic__, Delete__ or Rail__ before their names. These prefixes are used as a context for the parameters. You could do without this context, but then you would not have the option to for example have a length parameter under more than one handle type. This is because each parameter has to have a unique name and using these contexts is a neat way to get around that.</p>

<p>To add spare parameters, I created an extra function <code>addSpareParm()</code> in my Python Module. This function is added to the Callback Script of the &#8220;Add&#8221; Button. In this function, I retrieve the parameter that is currently selected in the &#8220;Spare Parms&#8221; menu. Next, I retrieve the default value for this parameter and lastly, I add this parameter to the active handle dictionary.</p>



<p>To delete spare parameters, I create an extra function <code>deleteSpareParm()</code> in my Python Module. This function will be called whenever the user presses the &#8220;Delete&#8221; button. In this function we can get the name of the parameter that should be deleted using <code>kwargs["parmtuple"].name()</code>. Afterwards, this parameter is removed from the active handle dictionary.</p>

<h5 class="has-background wp-block-heading" id="curvePosition" style="background-color:#2b2b2b">Curve Position</h5>
</div>



<div class="wp-block-column is-layout-flow wp-block-column-is-layout-flow"></div>
</div>



<p>The &#8220;curvePos&#8221; parameter defines at what arclength value along my racetrack the handle is positioned. This can be a value from 0 to the length of your track.  </p>



<p>You could also use a  &#8220;curveU&#8221; value which maps the entire curve between a value from 0 to 1. The downside to using a curveU value is that when you change the length of your curve that the handles will jump around as their position is defined as a percentage of the entire length of the curve. You won&#8217;t have this when you define the position as an arclength value.</p>



<figure class="wp-block-image size-full is-resized m0"><img decoding="async" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image_2022-01-23_164626.png" alt="" class="wp-image-188" width="500" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_164626.png 760w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_164626-288x300.png 288w" sizes="(max-width: 760px) 100vw, 760px" /></figure>



<p class="m0">I created a <code>getCurvePos()</code> function that gets a list of all the &#8220;curvePos&#8221; values of all the handles that are in the json. To get the index we can just sort the list and get the index by value. This index represents the position in the json list in which our handle dictionary should be placed. </p>



<div class="wp-block-columns m0 is-layout-flex wp-container-core-columns-is-layout-9d6595d7 wp-block-columns-is-layout-flex">

<h5 class="has-background wp-block-heading" id="changing-the-curve-position" style="background-color:#2b2b2b">Changing the curve position</h5>
</div>



<div class="wp-block-column is-layout-flow wp-block-column-is-layout-flow"></div>
</div>



<p>By changing the &#8220;curvePos&#8221; parameter the corresponding handle will move along. The handles are all ordered in the json list based on their &#8220;curvePos&#8221; value. So if you change the &#8220;curvePos&#8221; value then it could be that this order is not correct anymore. To check this I added a function to my Python Module called <code>updateCurvePos()</code> which I set in the Callback Script of the &#8220;curvePos&#8221; parameter. This function will call the <code>getCurvePos()</code> <a href="#curvePosition">function </a>to get the correct index of this handle. If the index is not equal to the current index of the handle dictionary in the list, then the handle dictionary is removed from the list and inserted back at the correct index.</p>

<p class="m0">Now that the setup is done we still need a way to change the active parameter and cycle through the list we just created. <figure class="wp-block-image size-full is-resized"><img decoding="async" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image_2022-01-23_200210.png" alt="" class="wp-image-257" width="500" title="m0" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_200210.png 961w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_200210-300x27.png 300w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_200210-768x68.png 768w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_200210-950x85.png 950w" sizes="(max-width: 961px) 100vw, 961px" /></figure>

<p class="m0">Now when you scroll with your MMB, the <code>cycleMode()</code> function will be called. In this function it is just a matter of getting the active parameter &gt; getting the index of this parameter in the  cyclable_active_parms list. Adding the direction to this index &gt; getting the new active parameter from the list &gt; and lastly setting the active parameter to this new one. After that we can cycle through all the cyclable_active_parms, disable all of them and only enable the active parameter.</p>



<figure class="wp-block-image size-full is-resized m0"><img decoding="async" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image_2022-01-23_202145.png" alt="" class="wp-image-265" width="500" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_202145.png 552w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_202145-300x52.png 300w" sizes="(max-width: 552px) 100vw, 552px" /></figure>

<h4 class="has-background wp-block-heading" id="updating-parameter-values" style="background-color:#333333">Updating parameter values</h4>



<p>Updating the parameter value by holding LMB and dragging is a bit complexer. Again, tart by getting some of the most important values we can get from the onMouseEvent(). </p>



<figure class="wp-block-image size-full is-resized m0"><img decoding="async" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image-11.png" alt="" class="wp-image-239" width="500" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image-11.png 655w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image-11-300x128.png 300w" sizes="(max-width: 655px) 100vw, 655px" /></figure>



<p>You will still need to add the self.mouse_x_start variable to your class. You can set it using <code>device.mouseX()</code> when the reason is &#8220;Start&#8221;. If the reason is &#8220;Active&#8221; then this means the user is holding LMB and dragging their mouse. When this happens, we want to update our parameter. Let&#8217;s get the old value, add the direction and set the new value. </p>



<figure class="wp-block-image size-full is-resized m0"><img decoding="async" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image_2022-01-23_190711.png" alt="" class="wp-image-242" width="500" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_190711.png 516w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-23_190711-300x84.png 300w" sizes="(max-width: 516px) 100vw, 516px" /></figure>



<h4 class="has-background wp-block-heading" id="multiplier-tag" style="background-color:#2b2b2b">Multiplier tag</h4>



<p class="m0">At the moment the rate at which the parameter value changes is always one. However, some parameters need to go quicker up and down than others. To define the multiplier for a certain parameter, change the active_parm_cycle tag so it is structured as a dictionary with the multiplier being a key-value pair.</p>



<figure class="wp-block-image size-full m0"><img loading="lazy" decoding="async" width="343" height="26" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image-22.png" alt="" class="wp-image-404" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image-22.png 343w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image-22-300x23.png 300w" sizes="auto, (max-width: 343px) 100vw, 343px" /></figure>



<p>Next in the python viewer state we have to create a new class variable called <code>self.active_parm_tags</code>. Each time the user enters the state or changes the active parameter, we will call the following function that will set this variable.</p>



<figure class="wp-block-image size-full is-resized m0"><img decoding="async" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image-23.png" alt="" class="wp-image-405" width="500" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image-23.png 787w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image-23-300x78.png 300w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image-23-768x199.png 768w" sizes="(max-width: 787px) 100vw, 787px" /></figure>



<p>If we now calculate the <code>new_value</code> we can multiple the <code>direction</code> with the multiplier defined in the <code>self.active_parm_tags</code> variable. Below, I am changing these values purely by scrolling with my mouse. Note how they all change with different steps. This is because of the multiplier. This way we can also change the handle type through our viewer state if we set the multiplier to one. I also added the functionality to change the value by scrolling with your middle mouse button. Use device.mouseWheel() in the onMouseWheelEvent() to get the scroll direction.</p>

<h4 class="has-background wp-block-heading" id="adding-and-deleting-spare-parameters" style="background-color:#2b2b2b">Adding and deleting spare parameters</h4>



<p class="m0">To do this we need to create some extra keybinds in our class. Add an <code>addSpareParm()</code> and a <code>deleteSpareParm()</code> function to the viewer state. To add a spare parameter, just call the <code>addSpareParm()</code> <a href="#spareParms">function </a>from the python module. </p>



<p>To delete a spare parameter first get the currently active parameter and check if this parameter has the &#8220;spare_parm = 1&#8221; tag. If this is the case then pass the parameter name on to the <code>deleteSpareParm()</code> function in the Python module to delete it.</p>



<figure class="wp-block-image size-full m0"><img loading="lazy" decoding="async" width="468" height="29" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image_2022-01-24_141924.png" alt="" class="wp-image-419" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-24_141924.png 468w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-24_141924-300x19.png 300w" sizes="auto, (max-width: 468px) 100vw, 468px" /></figure>



<h4 class="has-background wp-block-heading" id="creating-a-hud-info-panel" style="background-color:#2b2b2b">Geometry functions</h4>



<p>I do this by using something I made myself called geometry functions. </p>



<p>The idea is that using python viewer states we click in our viewport and get the position of where we clicked by colliding with a collision geometry (<a href="#selecting-handles-in-the-viewport">see selecting handles</a>). This position is stored in a vector parameter in our user interface. This is the input of our geometry function. This position is read into the geometry network and a point is created at this position. I ray the point onto my racetrack curve from which I can get the &#8220;curvePos&#8221; value. All of the values that I want to use as output for my geometry function I store in a parameter on a single output node. This way I can set the output parameters in the user interface by just grabbing the value from this output node.</p>



<p>To use this geometry function you first have to enter a special mode called &#8220;SetParm&#8221; using a keyboard shortcut. When you are in this mode, then every mouse action in the viewport will run our special geometry function.</p>



<figure class="wp-block-image size-large"><img loading="lazy" decoding="async" width="1024" height="355" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/GeometryFunctions-1024x355.png" alt="" class="wp-image-553" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/GeometryFunctions-1024x355.png 1024w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/GeometryFunctions-300x104.png 300w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/GeometryFunctions-768x266.png 768w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/GeometryFunctions-1536x532.png 1536w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/GeometryFunctions-2048x710.png 2048w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/GeometryFunctions-2000x693.png 2000w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></figure>

The value consists of three parts. The first one being which geometry function to use. The second is which collision geometry to use. This value is used in another geometry function to switch the collision geometry in my geometry network. And the last part is which output to use.</p>

<h1 class="alignwide has-text-align-left has-luminous-vivid-orange-background-color has-background has-medium-font-size wp-block-heading" id="integration-into-unreal-engine">Integration into Unreal Engine</h1>



<p>To integrate a tool like this into Unreal Engine you will need to install the Houdini Engine for Unreal plugin. This will allow you to import your Houdini digital assets into Unreal and Unreal will recognize them. The most important parameter on my HDA is the curve input. </p>



<figure class="wp-block-image size-full is-resized m0"><img decoding="async" src="https://jeroen.denayer.com/wp-content/uploads/2022/01/image_2022-01-27_150229.png" alt="" class="wp-image-578" width="500" srcset="https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-27_150229.png 690w, https://jeroen.denayer.com/wordpress/wp-content/uploads/2022/01/image_2022-01-27_150229-252x300.png 252w" sizes="(max-width: 690px) 100vw, 690px" /></figure>



<p>I create the shape of the racetrack inside of Unreal using this curve. Afterwards, I open a live connection between Unreal and Houdini using Houdini session sync. Now I am in Houdini and I can start editing my racetrack using the  python viewer state that I created.</p>



<h1 class="alignwide has-text-align-left has-luminous-vivid-orange-background-color has-background has-medium-font-size wp-block-heading" id="prototyping">Reflection</h1>



<h4 class="wp-block-heading" id="so-how-did-the-tool-compare-when-built-with-and-without-python-viewer-states-how-did-i-experience-using-python-in-houdini">So how did the tool compare when built with and without python viewer states? How did I experience using Python in Houdini? </h4>



<p>Well, it was not easy, but still I think it was worth investing my time researching this. I really believe that I created an interactive, user-friendly tool that can benefit a game company if they would want to create a racetrack. Of course my tool can still be refined but that&#8217;s not what this research was about. Creating the tool is the hard part. Adding extra options and increasing the graphical quality is the &#8220;relatively&#8221; easy part. And that is something that I might consider doing in the future. </p>



<p>While creating this tool I learned so much about Python in Houdini. One of the most important things I learned was that you can literally connect every part of Houdini together. At times I felt like I was wrongly using Houdini and Python. For example with the parameter tags to define geometry functions. It almost felt like hacking or abusing the system. But who is to say what the correct way of doing things is? If it is possible and it works, then that is the only thing that matters. </p>



<p>I do have to mention that you can clearly feel that the part of the Houdini object model (HOM) that is used to work with Houdini digital assets is not yet perfect for creating a super interactive, flawless user interface. Quite often I came across bugs or couldn&#8217;t find functions to do specific things and I had to find workarounds. If the workaround does its job then you could say that there is no problem at all. However, I would tend to disagree because these workarounds lead to lost time for the developer of the tool and make it more difficult to maintain.</p>



<p>But, this is also part of the process. Houdini and the python integration are in constant development. The only thing that we can do as users of Houdini is give feedback. All the 14 bugs I found I sent to SideFx support. I also sent 5 feature requests of which 3 have already been accepted. When I could not find how to do something I first would experiment, watch tutorials or ask around on forums. When that didn&#8217;t work the collaboration with SideFx support went very well.</p>



<h4 class="wp-block-heading" id="is-using-houdini-engine-in-combination-with-houdini-session-sync-a-benefit-to-the-game-asset-creation-pipeline">Is using Houdini engine in combination with Houdini session sync a benefit to the game asset creation pipeline?</h4>



<p>There is always the valid recommendation to use as little software as possible when creating a game. This drastically simplifies the pipeline. The more you can do inside of your game engine directly the better. However, I believe that this is not the case when these softwares are limiting your possibilities for creating the best game that you can possibly create. Houdini Engine and Houdini session sync are really good tools to link Unreal Engine and Houdini together. They allow you to tap into all the things that Houdini has to offer. There are still flaws and you will even have crashes, but I am confident this will get better over time. And in general I think that that these tools are way more valuable than a few rough edges here and there.</p>



<h4 class="wp-block-heading" id="in-general">In general</h4>



<p>Bridging the gap between Houdini and Unreal complicates the pipeline and artists will have to learn to work with Houdini and the Houdini tools that tool developers create for them. But, it is absolutely worth to use Houdini. The software is super powerful and flexible as I have demonstrated with my racetrack tool. It is no wonder why more and more game companies are starting to make use of Houdini and all of the benefits that proceduralism brings with it. For me personally the end goal is always to make as much quality content in as little time as possible and make this content available in a game engine as well. Using the combination of Houdini, Houdini Engine and Houdini session sync I am able to do this. I am not paid to say this. I really believe that Houdini will play a bigger and bigger role in the future of game development. In the meantime I will keep learning; keep exploring Python and Houdini and I hope that you could take something away from my research.</p>
